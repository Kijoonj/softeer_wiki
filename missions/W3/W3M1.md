# W3M1
- Choosing the base image for dockerfile
    - 도커 파일 작성시, 베이스 이미지로 apache/hadoop:3.3.6을 바로 가져올 수도 있고, ubuntu에서 시작하고 RUN command를 통해 따로 하둡을 설치할 수도 있다.
    - 전자가 편의성은 좋겠지만, 필요한 dependency를 파악하고, 환경변수를 직접 세팅하는 편이 배우는 것이 많을 것 같아 이번 미션은 후자를 선택한다.
- Setting ENV:
    - PATH파일에 기존 PATH variable($PATH) + $HADOOP_HOME/bin 을 추가
    
    ```jsx
    PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
    ```
    
- JDK, SSH, Hadoop 등 필요한 dependencies들을 설치한다.
- Hadoop Configuration ; .xml config파일을 사용한다
    - core-site.xml : hdfs의 namenode의 주소를 세팅한다.
    - hdfs-site.xml : dfs.replication을 통해 replication factor를 설정한다. dfs.namenode.name.dir를 통해 네임노드의 디렉토리를 설정한다.
    - mapred-site.xml :
- boostrap.sh
    - 처음 컨테이너 동작시 필요한 동작들을 정리해야한다.
    - hdfs namenode -format: 클러스터 초기 세팅시 단 1번만 실행하며 NameNode가 datanode들에 대한 정보를 정리할 수 있게한다.
    - start-dfs.sh:  /sbin 에 위치하며, NameNode에서 실행하여 namenode, datanode daemon을 실행시킨다.
- Docker build
    - -t를 사용하여 이미지 이름과 태그를 동시에 지정하여 빌드
